# startup.ts

Purpose: handle startup greeting and lifecycle footer status.

Responsibilities:

- randomized heraldic startup rapport via `WALDEMAR_STARTUP_REPORTS`
- recent campaign count in the greeting
- setup hint when settings are not applied
- randomized Waldemar footer status moods

Change `WALDEMAR_STARTUP_REPORTS` here to alter welcome rapports. Change `WALDEMAR_STATUS_MOODS` to alter footer messages and emoji.
